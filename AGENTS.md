# 外卖平台开发助手指南（Food Delivery App Agent Guide）

你正在开发一个外卖网站的 MVP（最小可用版本）。

## 目标
构建一个可用的外卖平台，包含：
- 顾客（customer）
- 商家（merchant）
- 管理员（admin）

## 技术栈
- 前端：Next.js
- 后端：FastAPI
- 数据库：PostgreSQL

## 核心模块
必须实现：
- 用户（user）
- 餐厅（restaurant）
- 菜单（menu）
- 购物车（cart）
- 订单（order）

## 工作流程（必须遵守）
1. 在编写代码前，先输出项目结构设计
2. 区分 MVP 功能 和 非 MVP 功能
3. 前后端必须清晰分离
4. 优先写清晰、易维护的代码，避免过度设计
5. 每完成一个阶段，更新 README
6. 在完成前，至少做基本验证或测试
